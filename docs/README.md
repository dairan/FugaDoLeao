# Docs - Foge Leao

Documentacao curta para manter o projeto facil de retomar.

## Ordem de leitura

1. `game-overview.md`: conceito, pilares e loop jogavel.
2. `architecture.md`: estrutura tecnica atual e contratos entre scripts/cenas.
3. `refactor-plan.md`: plano de refatoracao por fases.
4. `publishing-plan.md`: plano de publicacao por plataforma.
5. `public-release-checklist.md`: checklist para abrir o repositorio sem vazar dados.
6. `obsidian-workflow.md`: configuracao semantica e plugins recomendados.
7. `runbook.md`: comandos de validacao e fluxo local.
8. `git-workflow.md`: pratica de branches e commits.
9. `roadmap.md`: proximos passos recomendados.

## Principio

Este e um prototipo. A documentacao deve ajudar a tomar decisoes e mexer rapido, nao virar burocracia.
